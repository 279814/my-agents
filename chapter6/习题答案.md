### 习题1：AutoGen 与 LangGraph 深度对比

#### 协作模式对比

| 维度 | AutoGen | LangGraph |
|------|---------|----------|
| **基础架构** | 基于对话（Conversation-based） | 基于图（Graph-based） |
| **协作机制** | 通过群聊（GroupChat）实现多智能体通信，支持 RoundRobinGroupChat、SelectorGroupChat 等模式 | 通过状态图定义任务流程，节点间通过状态传递通信 |
| **消息传递** | 支持同步/异步消息传递，消息可包含文本、结构化数据 | 状态通过边流动，每个节点接收当前状态并返回新状态 |
| **对话管理** | 内置对话历史管理，支持多种终止条件（TextMentionTermination、HapticTermination） | 需要手动在状态中管理消息历史 |

#### 控制方式对比

| 维度 | AutoGen | LangGraph |
|------|---------|----------|
| **流程控制** | 声明式 + 策略驱动 | 状态机 + 条件边 |
| **动态路由** | 通过 Selector 选择下一个发言者 | 通过条件函数决定下一步节点 |
| **人机交互** | 支持 HumanInputBlock 实现人工介入 | 支持 interrupt 节点实现 human-in-the-loop |
| **容错机制** | 通过终止条件和最大轮次限制 | 通过条件边实现循环和回退 |

#### 适用场景对比

| 场景 | AutoGen | LangGraph |
|------|---------|----------|
| **软件开发团队** | ✅ 非常适合，多智能体角色扮演 | ⚠️ 需要较多自定义 |
| **对话式助手** | ✅ 适合，需要多轮交互 | ✅ 适合，流程清晰 |
| **复杂工作流编排** | ⚠️ 可以，但非最优 | ✅ 非常适合 |
| **Research 代理** | ✅ 适合深度讨论 | ✅ 适合多步骤研究 |
| **实时游戏** | ⚠️ 需要额外配置 | ⚠️ 需要额外配置 |

### 习题2：涌现式协作 vs 显式控制

#### 概念解释

**涌现式协作（Emergent Collaboration）**

涌现式协作是一种自下而上的设计哲学，智能体通过简单的交互规则自然产生复杂的行为模式。就像鸟群通过简单的跟随规则产生壮观的群体飞行图案一样，智能体也能通过局部交互产生全局优化的问题解决方案。

**显式控制（Explicit Control）**

显式控制是一种自上而下的设计哲学，所有智能体的行为都由明确的规则和流程定义。就像交响乐演奏需要指挥家的精确控制一样，显式控制系统中的每个步骤都是预先定义和可预测的。

#### 核心权衡

```
涌现式协作                          显式控制
    │                                 │
    ├── 优点：                        ├── 优点：
    │   • 更高的灵活性                   │   • 可预测性和确定性
    │   • 可能的创新性解决方案             │   • 易于调试和验证
    │   • 更接近人类协作方式              │   • 流程可审计追溯
    │   • 适应未知/模糊任务               │   • 错误恢复容易
    │                                 │
    ├── 缺点：                        ├── 缺点：
    │   • 难以预测和调试                  │   • 灵活性较低
    │   • 可能陷入循环或无效状态            │   • 难以处理新情况
    │   • 难以保证结果质量                │   • 可能过度工程化
    │   • 资源消耗不可控                  │   • 缺乏创造性
    └── 权衡点：                       └── 权衡点：
        不确定性 ↔ 可控性                    灵活性 ↔ 规范性
```

#### 实践中的平衡策略

### 习题3：动态回退机制设计

#### 问题分析

当前 RoundRobinGroupChat 模式是固定顺序轮换，无法满足"代码审查不通过需返回产品经理"的动态回退需求。

#### 设计方案

```python
from enum import Enum
from autogen_agentchat.agents import AssistantAgent
from autogen_agentchat.base import TerminationCondition
from autogen_agentchat.teams import GroupChat
from autogen_agentchat.ui import Console
from typing import Annotated
from dataclasses import dataclass

class WorkFlowState(Enum):
    """工作流状态枚举"""
    REQUIREMENTS = "requirements"
    DESIGN = "design"
    IMPLEMENTATION = "implementation"
    CODE_REVIEW = "code_review"
    TESTING = "testing"
    APPROVAL = "approval"
    REVISION = "revision"  # 新增：需要修订的状态

class DynamicFallbackGroupChat(GroupChat):
    """
    支持动态回退的群聊实现

    核心机制：
    1. 维护一个工作流状态机
    2. 根据审查结果决定是否回退
    3. 动态调整下一个发言者
    """

    def __init__(
        self,
        participants: list,
        workflow_state: WorkFlowState = WorkFlowState.REQUIREMENTS,
        **kwargs
    ):
        super().__init__(participants, **kwargs)
        self.workflow_state = workflow_state
        self.state_history = [workflow_state]
        self.max_revisions = 3  # 最大回退次数

        # 定义状态转换规则
        self.transitions = {
            WorkFlowState.REQUIREMENTS: [WorkFlowState.DESIGN],
            WorkFlowState.DESIGN: [WorkFlowState.IMPLEMENTATION],
            WorkFlowState.IMPLEMENTATION: [WorkFlowState.CODE_REVIEW],
            WorkFlowState.CODE_REVIEW: [
                WorkFlowState.TESTING,      # 审查通过
                WorkFlowState.REVISION,     # 需要修订
            ],
            WorkFlowState.REVISION: [
                WorkFlowState.IMPLEMENTATION,  # 返回实现
                WorkFlowState.REQUIREMENTS,    # 返回需求（重大变更）
            ],
            WorkFlowState.TESTING: [WorkFlowState.APPROVAL],
            WorkFlowState.APPROVAL: [WorkFlowState.TESTING, END],  # 可回退测试或结束
        }

        # 状态与角色的映射
        self.state_to_roles = {
            WorkFlowState.REQUIREMENTS: ["product_manager"],
            WorkFlowState.DESIGN: ["architect"],
            WorkFlowState.IMPLEMENTATION: ["engineer"],
            WorkFlowState.CODE_REVIEW: ["code_reviewer", "engineer"],  # 多人审查
            WorkFlowState.REVISION: ["engineer", "product_manager"],
            WorkFlowState.TESTING: ["qa_engineer"],
            WorkFlowState.APPROVAL: ["product_manager", "tech_lead"],
        }

    def transition_to(self, new_state: WorkFlowState) -> bool:
        """执行状态转换"""
        if new_state not in self.transitions[self.workflow_state]:
            return False

        # 检查回退次数限制
        if new_state in [WorkFlowState.REVISION, WorkFlowState.REQUIREMENTS]:
            revision_count = sum(1 for s in self.state_history if s == WorkFlowState.REVISION)
            if revision_count >= self.max_revisions:
                print(f"警告：已达到最大回退次数 ({self.max_revisions})")
                return False

        self.workflow_state = new_state
        self.state_history.append(new_state)
        print(f"状态转换: -> {new_state.value}")
        return True

    def get_next_speaker(self) -> str:
        """根据当前状态确定下一个发言者"""
        eligible_roles = self.state_to_roles.get(self.workflow_state, [])

        # 找到第一个匹配角色的发言者
        for participant in self.participants:
            if participant.name in eligible_roles:
                return participant.name

        # 默认轮换
        current_index = self.participants.index(self.current_speaker) if self.current_speaker else -1
        return self.participants[(current_index + 1) % len(self.participants)].name

    def should_fallback(self, review_result: str) -> bool:
        """根据审查结果判断是否需要回退"""
        fallback_keywords = ["需要修改", "不符合要求", "重新实现", "revise", "reject"]
        return any(keyword in review_result.lower() for keyword in fallback_keywords)

# 审查结果解析器
def parse_review_result(message: str) -> dict:
    """解析审查消息，提取审查结果"""
    return {
        "approved": "approve" in message.lower() and "reject" not in message.lower(),
        "needs_revision": any(kw in message.lower() for kw in ["需要修改", "revise", "修改"]),
        "revision_type": "implementation" if "代码" in message else "requirements",
        "reason": message
    }
```

### 习题5：对话质量监控机制设计

#### 系统架构

```python
from enum import Enum
from dataclasses import dataclass, field
from typing import Optional, Callable
import time
import re

class ConversationIssue(Enum):
    """对话异常类型枚举"""
    OFF_TOPIC = "off_topic"           # 偏离主题
    LOOP = "loop"                      # 陷入循环
    STALL = "stall"                    # 卡顿无进展
    INCOMPLETE = "incomplete"          # 回复不完整
    CONFLICT = "conflict"              # 持续冲突
    VERBOSE = "verbose"                # 过于冗长
    EMPTY = "empty"                    # 空回复

@dataclass
class QualityMetrics:
    """质量度量指标"""
    relevance_score: float = 1.0      # 主题相关性 (0-1)
    novelty_score: float = 1.0        # 信息新颖度 (0-1)
    coherence_score: float = 1.0      # 连贯性 (0-1)
    conciseness_score: float = 1.0    # 简洁性 (0-1)
    progress_score: float = 1.0       # 进展程度 (0-1)

@dataclass
class ConversationMonitor:
    """
    对话质量监控器

    监控机制：
    1. 主题相关性检测：确保对话不偏离核心任务
    2. 循环检测：检测重复内容模式
    3. 进展评估：评估对话是否持续推进
    4. 简洁性检查：识别冗长或空洞的回复
    """

    # 配置参数
    max_repetition_threshold: int = 3       # 最大重复次数
    off_topic_threshold: float = 0.3         # 偏离主题阈值
    stall_turns_threshold: int = 3           # 卡顿判定轮次
    verbose_length_threshold: int = 2000     # 冗长判定长度
    min_content_length: int = 10             # 最小内容长度

    # 内部状态
    message_history: list = field(default_factory=list)
    repetition_buffer: list = field(default_factory=list)
    turn_count: int = 0
    last_meaningful_turn: int = 0
    issue_counts: dict = field(default_factory=dict)

    def __post_init__(self):
        self.issue_counts = {issue: 0 for issue in ConversationIssue}

    def analyze_message(self, message: str, expected_topics: list[str]) -> QualityMetrics:
        """分析单条消息的质量"""
        metrics = QualityMetrics()

        # 1. 主题相关性分析
        metrics.relevance_score = self._compute_relevance(message, expected_topics)

        # 2. 简洁性分析
        metrics.conciseness_score = self._compute_conciseness(message)

        # 3. 连贯性分析（基于消息历史）
        metrics.coherence_score = self._compute_coherence(message)

        return metrics

    def _compute_relevance(self, message: str, topics: list[str]) -> float:
        """计算主题相关性"""
        if not topics:
            return 1.0

        message_lower = message.lower()
        topic_matches = sum(1 for topic in topics if topic.lower() in message_lower)
        return topic_matches / len(topics)

    def _compute_conciseness(self, message: str) -> float:
        """计算简洁性得分"""
        if len(message) < self.min_content_length:
            return 0.3
        if len(message) > self.verbose_length_threshold:
            return 0.5

        # 检查是否有意义的句子数量
        sentences = re.split(r'[.!?。！？]+', message)
        avg_sentence_length = sum(len(s.strip()) for s in sentences) / max(len(sentences), 1)

        if avg_sentence_length < 10:
            return 0.4
        elif avg_sentence_length > 100:
            return 0.6
        return 1.0

    def _compute_coherence(self, message: str) -> float:
        """计算与历史的连贯性"""
        if len(self.message_history) < 2:
            return 1.0

        # 简单的词汇重叠分析
        current_words = set(message.lower().split())
        prev_words = set(self.message_history[-1].lower().split())

        overlap = len(current_words & prev_words)
        total = len(current_words | prev_words)

        return overlap / total if total > 0 else 1.0

    def detect_issues(self, message: str, conversation_history: list[str]) -> list[ConversationIssue]:
        """检测对话异常"""
        issues = []

        # 1. 检测循环
        if self._detect_loop(message):
            issues.append(ConversationIssue.LOOP)
            self.issue_counts[ConversationIssue.LOOP] += 1

        # 2. 检测偏离主题
        if self._is_off_topic(message):
            issues.append(ConversationIssue.OFF_TOPIC)
            self.issue_counts[ConversationIssue.OFF_TOPIC] += 1

        # 3. 检测空回复
        if self._is_empty(message):
            issues.append(ConversationIssue.EMPTY)
            self.issue_counts[ConversationIssue.EMPTY] += 1

        # 4. 检测冗长
        if self._is_verbose(message):
            issues.append(ConversationIssue.VERBOSE)
            self.issue_counts[ConversationIssue.VERBOSE] += 1

        # 5. 检测卡顿（无实质进展）
        if self._detect_stall(conversation_history):
            issues.append(ConversationIssue.STALL)
            self.issue_counts[ConversationIssue.STALL] += 1

        return issues

    def _detect_loop(self, message: str) -> bool:
        """检测重复循环"""
        # 归一化消息进行比较
        normalized = self._normalize_message(message)

        # 检查是否在最近的消息中重复出现
        for prev_msg in self.message_history[-self.max_repetition_threshold:]:
            if self._normalize_message(prev_msg) == normalized:
                return True

        self.message_history.append(message)
        return False

    def _normalize_message(self, message: str) -> str:
        """归一化消息用于比较"""
        return re.sub(r'\s+', '', message.lower())[:100]  # 去除空白并截断

    def _is_off_topic(self, message: str) -> bool:
        """检测是否偏离主题"""
        # 基于关键词的简单检测
        off_topic_patterns = [
            r'^下面是一.*故事',
            r'^作为一个.*我没有',
            r'^让我想想.*其他.*话题',
        ]
        for pattern in off_topic_patterns:
            if re.search(pattern, message):
                return True
        return False

    def _is_empty(self, message: str) -> bool:
        """检测空回复"""
        stripped = message.strip()
        return len(stripped) < self.min_content_length or stripped in ['...', '好的', 'OK', '嗯']

    def _is_verbose(self, message: str) -> bool:
        """检测冗长回复"""
        return len(message) > self.verbose_length_threshold * 2

    def _detect_stall(self, history: list[str]) -> bool:
        """检测卡顿（多轮无实质进展）"""
        if len(history) < self.stall_turns_threshold * 2:
            return False

        recent = history[-self.stall_turns_threshold*2:]
        # 检查最近几轮是否有重复模式
        patterns = [self._normalize_message(m) for m in recent]
        unique_patterns = len(set(patterns))

        # 如果模式太少，说明可能卡顿
        return unique_patterns <= len(patterns) / 2

    def should_intervene(self) -> tuple[bool, Optional[str], Optional[ConversationIssue]]:
        """
        判断是否需要干预

        Returns:
            (是否干预, 干预消息, 主要问题类型)
        """
        # 严重问题：立即干预
        if self.issue_counts[ConversationIssue.EMPTY] >= 2:

        if self.issue_counts[ConversationIssue.OFF_TOPIC] >= 3:
            return True, "注意：讨论似乎偏离了原始主题。", ConversationIssue.OFF_TOPIC

        if self.issue_counts[ConversationIssue.VERBOSE] >= 2:
            return True, "回复较为冗长，请精简要点。", ConversationIssue.VERBOSE

        return False, None, None

    def reset(self):
        """重置监控状态"""
        self.message_history.clear()
        self.repetition_buffer.clear()
        self.turn_count = 0
        self.last_meaningful_turn = 0
        self.issue_counts = {issue: 0 for issue in ConversationIssue}
```

### 习题6：MsgHub 消息驱动架构优势

#### 消息驱动 vs 传统函数调用

```
传统函数调用                          消息驱动架构
    │                                    │
    │                                    │
 Engineer ──call()──▶ Critic            Engineer ──Msg()──▶ MsgHub
    │                          ◀──────────────│        │
    │                                    Critic ──Msg()──▶│
    │      ◀──────────────│                          │
    │                                    All Agents ◀──broadcast()──
    │                                                    │
    │                                                    ▼
    │                                              每个Agent
    │                                              收到消息
```

#### 核心优势

| 优势维度 | 消息驱动架构 | 传统函数调用 |
|---------|------------|-------------|
| **解耦程度** | 发送者和接收者完全解耦 | 直接依赖，紧密耦合 |
| **动态性** | 可在运行时动态添加/移除参与者 | 需要在编译时确定调用关系 |
| **并发支持** | 原生支持多个消息并行处理 | 需要额外同步机制 |
| **可观测性** | 消息流可集中监控和记录 | 调用链分散，难以追踪 |
| **扩展性** | 新增智能体无需修改现有代码 | 可能需要修改调用方 |
| **灵活性** | 支持一对多、多对一、多对多通信 | 通常一对一 |

#### 典型场景


#### 适用场景价值

| 场景 | 消息驱动价值 |
|------|------------|
| **实时游戏（如狼人杀）** | 多个玩家需要同时接收游戏状态更新，消息驱动天然支持广播 |
| **分布式系统** | 不同服务器上的智能体通过消息通信，松耦合便于部署 |
| **监控系统** | 所有事件汇总到消息中心，便于统一日志和告警 |

#### 消息顺序性实现示例

```python
import asyncio
from dataclasses import dataclass, field
from typing import Any
import time
import uuid

@dataclass
class OrderedMessage:
    """带顺序保证的消息"""
    sequence_num: int           # 全局序列号
    lamport_clock: int          # Lamport 时钟
    timestamp: float            # 本地时间戳
    sender_id: str             # 发送者ID
    message_type: str           # 消息类型
    payload: Any                # 消息内容
    message_id: str = field(default_factory=lambda: str(uuid.uuid4()))

class DistributedMsgHub:
    """
    分布式消息中心

    保证消息的：
    1. **顺序性**：通过 Lamport Clock + 序列号
    2. **一致性**：通过消息确认和重试
    3. **可靠性**：通过消息持久化和复制
    """

    def __init__(self, server_id: str):
        self.server_id = server_id
        self.lamport_clock = 0
        self.message_queue: asyncio.PriorityQueue = asyncio.PriorityQueue()
        self.sequence_counter = 0
        self.pending_messages: dict[str, OrderedMessage] = {}
        self.delivered_messages: set[str] = set()

    def update_clock(self, received_clock: int = None):
        """更新 Lamport 时钟"""
        if received_clock is not None:
            self.lamport_clock = max(self.lamport_clock, received_clock) + 1
        else:
            self.lamport_clock += 1
        return self.lamport_clock

    async def send(self, message_type: str, payload: Any, target_ids: list[str]):
        """发送消息"""
        self.update_clock()
        self.sequence_counter += 1

        msg = OrderedMessage(
            sequence_num=self.sequence_counter,
            lamport_clock=self.lamport_clock,
            timestamp=time.time(),
            sender_id=self.server_id,
            message_type=message_type,
            payload=payload,
        )

        # 广播给所有目标服务器
        await self._broadcast_to_servers(msg, target_ids)

        # 等待确认
        await self._wait_for_ack(msg.message_id)

    async def receive(self, timeout: float = None) -> OrderedMessage:
        """接收消息，按顺序处理"""
        try:
            # 优先队列按 (sequence_num, lamport_clock) 排序
            msg = await asyncio.wait_for(
                self.message_queue.get(),
                timeout=timeout
            )

            # 去除重复
            if msg.message_id in self.delivered_messages:
                return await self.receive(timeout=timeout)

            self.delivered_messages.add(msg.message_id)

            # 更新本地时钟
            self.update_clock(msg.lamport_clock)

            return msg

        except asyncio.TimeoutError:
            return None

    async def _broadcast_to_servers(self, msg: OrderedMessage, target_ids: list[str]):
        """广播消息到其他服务器"""
        # 实际实现中会使用 RPC 或消息队列
        for target_id in target_ids:
            await self._send_to_server(msg, target_id)

    async def _wait_for_ack(self, message_id: str, timeout: float = 5.0):
        """等待消息确认"""
        start = time.time()
        while time.time() - start < timeout:
            if message_id in self.delivered_messages:
                return True
            await asyncio.sleep(0.01)
        return False

    def get_message_order_key(self, msg: OrderedMessage) -> tuple:
        """获取消息排序键"""
        return (msg.sequence_num, msg.lamport_clock)
```

#### 冲突解决机制

```python
from enum import Enum
from dataclasses import dataclass
from typing import Optional
import re

class ConflictType(Enum):
    """冲突类型"""
    TERMINATION_DISPUTE = "termination_dispute"    # 终止争议
    DIRECTION_DISPUTE = "direction_dispute"        # 方向争议
    QUALITY_DISPUTE = "quality_dispute"            # 质量争议
    PRIORITY_DISPUTE = "priority_dispute"          # 优先级争议

class ConflictStatus(Enum):
    """冲突状态"""
    NONE = "none"
    DETECTED = "detected"
    NEGOTIATING = "negotiating"
    RESOLVED = "resolved"
    ESCALATED = "escalated"

@dataclass
class ConflictRecord:
    """冲突记录"""
    conflict_type: ConflictType
    agent_a_position: str
    agent_b_position: str
    round: int
    evidence: list[str]
    resolution: Optional[str] = None

class ConflictResolver:
    """
    冲突解决器

    策略：
    1. 检测：识别冲突类型和证据
    2. 评估：评估冲突严重程度
    3. 协商：通过对话缩小分歧
    4. 仲裁：引入外部标准或投票
    5. 升级：在无法解决时升级处理
    """

    def __init__(self, threshold_agreement: float = 0.7):
        self.threshold_agreement = threshold_agreement
        self.current_conflict: Optional[ConflictRecord] = None
        self.conflict_history: list[ConflictRecord] = []
        self.max_negotiation_rounds = 5

    def detect_conflict(
        self,
        agent_a_response: str,
        agent_b_response: str,
        termination_marker: str = "<CAMEL_TASK_DONE>"
    ) -> Optional[ConflictRecord]:
        """检测冲突"""

        # 检查是否有关于终止的争议
        a_thinks_done = termination_marker in agent_a_response
        b_thinks_done = termination_marker in agent_b_response

        if a_thinks_done != b_thinks_done:
            self.current_conflict = ConflictRecord(
                conflict_type=ConflictType.TERMINATION_DISPUTE,
                agent_a_position="任务可以终止" if a_thinks_done else "任务需要继续",
                agent_b_position="任务可以终止" if b_thinks_done else "任务需要继续",
                round=1,
                evidence=[agent_a_response[:200], agent_b_response[:200]]
            )
            return self.current_conflict

        return None

    def resolve_termination_conflict(
        self,
        conflict: ConflictRecord,
        context_summary: str
    ) -> dict:
        """
        解决终止争议

        策略：
        1. 提取双方核心论点
        2. 基于质量标准评估
        3. 进行加权投票
        4. 决定是否终止
        """

        resolution = {
            "should_terminate": False,
            "reason": "",
            "remaining_work": [],
            "next_action": ""
        }

        # 评估任务完成度
        completion_score = self._evaluate_completion(context_summary)

        # 检查是否有明确未完成项
        incomplete_items = self._find_incomplete_items(context_summary)

        # 冲突解决逻辑
        if completion_score >= 0.9 and len(incomplete_items) == 0:
            # 高完成度，可以终止
            resolution["should_terminate"] = True
            resolution["reason"] = f"任务完成度达到 {completion_score*100}%，无明显遗留问题"
            resolution["next_action"] = "terminate"

        elif completion_score >= 0.7 and len(incomplete_items) <= 2:
            # 中等完成度，完成核心部分
            resolution["should_terminate"] = True
            resolution["reason"] = f"核心功能已完成 ({completion_score*100}%)，非关键项可后续处理"
            resolution["remaining_work"] = incomplete_items
            resolution["next_action"] = "terminate_with_notes"

        elif completion_score < 0.5:
            # 完成度不足，必须继续
            resolution["should_terminate"] = False
            resolution["reason"] = f"任务完成度仅 {completion_score*100}%，需要继续工作"
            resolution["remaining_work"] = incomplete_items
            resolution["next_action"] = "continue"

        else:
            # 分歧较大，进入协商
            negotiation_result = self._negotiate(conflict)
            return negotiation_result

        return resolution

    def _evaluate_completion(self, context: str) -> float:
        """评估任务完成度"""
        # 基于关键词的出现情况评估
        completion_indicators = [
            "完成", "实现", "解决了", "达到了",
            "completed", "implemented", "achieved", "finished"
        ]

        indicator_count = sum(1 for ind in completion_indicators if ind in context)
        base_score = min(indicator_count * 0.15, 0.6)

        # 加上基础分
        return min(base_score + 0.3, 1.0)

    def _find_incomplete_items(self, context: str) -> list[str]:
        """识别未完成项"""
        incomplete_patterns = [
            r"还需要(.+)",
            r"待(.+)",
            r"未完成(.+)",
            r"应该(.+)但没有",
            r"pending(.+)",
            r"TODO[:\s]+(.+)",
        ]

        items = []
        for pattern in incomplete_patterns:
            matches = re.findall(pattern, context, re.IGNORECASE)
            items.extend(matches)

        return items[:5]  # 最多返回5项

    def _negotiate(self, conflict: ConflictRecord) -> dict:
        """协商解决"""
        if conflict.round >= self.max_negotiation_rounds:
            return self._escalate(conflict)

        # 更新协商轮次
        conflict.round += 1

        # 生成协商建议
        suggestion = self._generate_compromise_suggestion(conflict)

        return {
            "should_terminate": None,  # 表示需要继续协商
            "reason": suggestion,
            "round": conflict.round,
            "next_action": "negotiate"
        }

    def _generate_compromise_suggestion(self, conflict: ConflictRecord) -> str:
        """生成妥协建议"""
        return f"""建议方案：
1. 双方各自列出已完成的工作和待完成的工作
2. 优先级排序：哪些是必须的，哪些是可选的
3. 如果核心功能已完成，可以考虑标记任务为"基本完成"
4. 非核心项可以放到后续迭代中处理"""

    def _escalate(self, conflict: ConflictRecord) -> dict:
        """升级处理"""
        conflict_history = self.conflict_history

        # 记录冲突
        self.conflict_history.append(conflict)
        self.current_conflict = None

        # 强制终止，但记录争议
        return {
            "should_terminate": True,
            "reason": f"经过 {self.max_negotiation_rounds} 轮协商无法达成一致，强制终止",
            "conflict_recorded": True,
            "next_action": "force_terminate_with_escalation"
        }
```

#### 集成到 CAMEL Society

```python
class ConflictAwareSociety(RolePlaying):
    """
    支持冲突解决的 CAMEL Society 扩展
    """

    def __init__(self, *args, conflict_resolver: ConflictResolver = None, **kwargs):
        super().__init__(*args, **kwargs)
        self.conflict_resolver = conflict_resolver or ConflictResolver()
        self.termination_disputes = 0

    def step(self, input_message: dict) -> tuple:
        """执行一步，带冲突检测"""
        assistant_response, user_response = super().step(input_message)

        # 检测冲突
        conflict = self.conflict_resolver.detect_conflict(
            assistant_response.msg.content,
            user_response.msg.content
        )

        if conflict:
            self.termination_disputes += 1

            # 尝试解决冲突
            context = self._build_context_summary()
            resolution = self.conflict_resolver.resolve_termination_conflict(
                conflict, context
            )

            if resolution["next_action"] == "force_terminate_with_escalation":
                # 强制终止
                return assistant_response, user_response, resolution

            elif resolution["next_action"] == "negotiate":
                # 继续协商
                input_message = resolution["reason"]

        return assistant_response, user_response, None

    def _build_context_summary(self) -> str:
        """构建上下文摘要"""
        # 汇总对话历史
        messages = [getattr(msg, 'content', str(msg)) for msg in self.messages]
        return " ".join(messages[-10:])
```

```python
from enum import Enum

class QualityLevel(Enum):
    HIGH = "high"
    MEDIUM = "medium"
    LOW = "low"

def evaluate_quality(state: QASState) -> QualityLevel:
    """评估答案质量"""
    answer = state.get("answer", "")

    # 质量评估指标
    metrics = {
        "length_score": len(answer) / 500 if len(answer) < 2000 else 1.0,  # 长度
        "detail_score": min(len(answer.split('.')), 10) / 10,  # 细节程度
        "specificity_score": 1.0 if any(kw in answer for kw in ["具体", "例如", "比如", "具体来说"]) else 0.5,  # 具体性
    }

    # 综合得分
    overall_score = sum(metrics.values()) / len(metrics)

    if overall_score >= 0.8:
        return QualityLevel.HIGH
    elif overall_score >= 0.5:
        return QualityLevel.MEDIUM
    else:
        return QualityLevel.LOW

def should_reflect(state: QASState) -> str:
    """决定是否需要反思"""
    quality = evaluate_quality(state)

    if quality == QualityLevel.HIGH:
        return "high"  # 直接结束
    elif quality == QualityLevel.MEDIUM:
        return "medium"  # 可以优化，但不强制
    else:
        return "low"  # 必须重新生成

workflow.add_node("rewrite_query", rewrite_query_node)

    "answer",
    should_reflect,
    {

        "medium": END,  # 也可以选择优化
        "low": "reflect"
    }
)

workflow.add_edge("rewrite_query", "new_search")
workflow.add_edge("new_search", "answer")

    max_reflections: int = 2

        # 达到最大反思次数，强制结束
        return state

        **state,
```

**架构设计：**

```python
from langgraph.graph import StateGraph, END, START
from dataclasses import dataclass
from typing import Optional
from datetime import datetime

@dataclass
class ApprovalState:
    """审批流程状态"""
    # 申请人信息
    applicant_id: str
    application_type: str
    submitted_at: datetime

    # 各阶段结果
    document_check: dict = None  # {"passed": bool, "reason": str, "checked_at": datetime}
    risk_assessment: dict = None  # {"score": float, "level": str, "factors": list}
    amount_calculated: float = 0.0
    compliance_check: dict = None  # {"passed": bool, "violations": list}
    manual_review: dict = None  # {"reviewer": str, "decision": str, "comments": str}
    final_decision: str = ""

    # 审计追踪
    audit_trail: list = None

    def __post_init__(self):
        if self.audit_trail is None:
            self.audit_trail = []
        if self.document_check is None:
            self.document_check = {}
        if self.risk_assessment is None:
            self.risk_assessment = {}
        if self.compliance_check is None:
            self.compliance_check = {}
        if self.manual_review is None:
            self.manual_review = {}

def document_check_node(state: ApprovalState) -> ApprovalState:
    """阶段1：资料审核"""
    # 执行资料审核逻辑
    result = check_documents(state.applicant_id)

    state.document_check = {
        "passed": result["valid"],
        "reason": result.get("reason", ""),
        "checked_at": datetime.now(),
        "checked_by": "system"
    }

    # 记录审计
    state.audit_trail.append({
        "stage": "document_check",
        "result": result,
        "timestamp": datetime.now()
    })

    return state

def risk_assessment_node(state: ApprovalState) -> ApprovalState:
    """阶段2：风险评估"""
    if not state.document_check.get("passed"):
        state.risk_assessment = {"score": 0, "level": "REJECTED", "reason": "资料审核未通过"}
        return state

    # 执行风险评估
    risk_result = assess_risk(state.applicant_id)

    state.risk_assessment = {
        "score": risk_result["score"],
        "level": risk_result["level"],  # LOW, MEDIUM, HIGH, VERY_HIGH
        "factors": risk_result["factors"],
        "assessed_at": datetime.now()
    }

    state.audit_trail.append({
        "stage": "risk_assessment",
        "result": risk_result,
        "timestamp": datetime.now()
    })

    return state

def amount_calculation_node(state: ApprovalState) -> ApprovalState:
    """阶段3：额度计算"""
    if state.risk_assessment.get("level") in ["HIGH", "VERY_HIGH"]:
        state.amount_calculated = 0  # 高风险不授信
        return state

    # 基于风险评分计算额度
    base_amount = calculate_base_amount(state.applicant_id)
    risk_multiplier = {
        "LOW": 1.0,
        "MEDIUM": 0.7,
        "HIGH": 0.3
    }.get(state.risk_assessment.get("level", "MEDIUM"), 0)

    state.amount_calculated = base_amount * risk_multiplier

    state.audit_trail.append({
        "stage": "amount_calculation",
        "result": {"amount": state.amount_calculated},
        "timestamp": datetime.now()
    })

    return state

def compliance_check_node(state: ApprovalState) -> ApprovalState:
    """阶段4：合规检查"""
    violations = check_compliance(state.applicant_id, state.amount_calculated)

    state.compliance_check = {
        "passed": len(violations) == 0,
        "violations": violations,
        "checked_at": datetime.now()
    }

    state.audit_trail.append({
        "stage": "compliance_check",
        "result": state.compliance_check,
        "timestamp": datetime.now()
    })

    return state

def manual_review_node(state: ApprovalState) -> ApprovalState:
    """阶段5：人工复核（可中断等待人工输入）"""
    # 在实际实现中，这里会暂停等待人工审核
    # 使用 LangGraph 的 interrupt 机制
    if state.amount_calculated > 100000:  # 大额需要人工
        # 这里会中断流程，等待人工输入
        # state = await interrupt(state)
        pass

    return state

def final_decision_node(state: ApprovalState) -> ApprovalState:
    """阶段6：最终决策"""
    # 综合所有阶段结果做出决策
    if not state.document_check.get("passed"):
        state.final_decision = "REJECTED"
    elif state.risk_assessment.get("level") in ["HIGH", "VERY_HIGH"]:
        state.final_decision = "REJECTED"
    elif not state.compliance_check.get("passed"):
        state.final_decision = "REJECTED"
    elif state.manual_review.get("decision") == "REJECTED":
        state.final_decision = "REJECTED"
    elif state.amount_calculated == 0:
        state.final_decision = "REJECTED"
    else:
        state.final_decision = "APPROVED"

    state.audit_trail.append({
        "stage": "final_decision",
        "result": {"decision": state.final_decision},
        "timestamp": datetime.now()
    })

    return state

# 条件边：定义分支逻辑
def should_proceed_to_risk(state: ApprovalState) -> str:
    """判断是否进入风险评估阶段"""
    return "proceed" if state.document_check.get("passed") else "reject"

def should_proceed_to_amount(state: ApprovalState) -> str:
    """判断是否进入额度计算阶段"""
    return "proceed" if state.risk_assessment.get("level") not in ["HIGH", "VERY_HIGH"] else "reject"

def should_proceed_to_compliance(state: ApprovalState) -> str:
    """判断是否进入合规检查阶段"""
    return "proceed" if state.amount_calculated > 0 else "reject"

def should_manual_review(state: ApprovalState) -> str:
    """判断是否需要人工复核"""
    return "required" if state.amount_calculated > 100000 else "skipped"

# 构建图
workflow = StateGraph(ApprovalState)

workflow.add_node("document_check", document_check_node)
workflow.add_node("risk_assessment", risk_assessment_node)
workflow.add_node("amount_calculation", amount_calculation_node)
workflow.add_node("compliance_check", compliance_check_node)
workflow.add_node("manual_review", manual_review_node)
workflow.add_node("final_decision", final_decision_node)

workflow.add_edge(START, "document_check")

workflow.add_conditional_edges(
    "document_check",
    should_proceed_to_risk,
    {"proceed": "risk_assessment", "reject": "final_decision"}
)

workflow.add_conditional_edges(
    "risk_assessment",
    should_proceed_to_amount,
    {"proceed": "amount_calculation", "reject": "final_decision"}
)

workflow.add_conditional_edges(
    "compliance_check",
    should_proceed_to_compliance,
    {"proceed": "compliance_check", "reject": "final_decision"}
)

workflow.add_conditional_edges(
    "manual_review",
    should_manual_review,
    {"required": "manual_review", "skipped": "final_decision"}
)

workflow.add_edge("final_decision", END)

# 编译图
graph = workflow.compile()

# 支持审计：获取完整执行路径
async def run_with_audit(application):
    # 执行主流程
    result = await graph.ainvoke(application)

    # 生成审计报告
    audit_report = {
        "application_id": application.applicant_id,
        "execution_path": [entry["stage"] for entry in result.audit_trail],
        "full_audit_trail": result.audit_trail,
        "final_decision": result.final_decision,
        "timestamp": datetime.now()
    }

    return result, audit_report
```

## 总结

### 框架选择决策矩阵

| 应用场景 | 推荐框架 | 关键理由 |
|---------|---------|---------|
| 高并发客服 (1000+ QPS) | LangGraph | 无状态、水平扩展、低延迟 |
| 深度协作写作 | CAMEL | 角色扮演、自动推进、双智能体协作 |
| 流程审批系统 | LangGraph | 状态机、审计追溯、human-in-the-loop |
| 软件开发团队 | AutoGen | GroupChat、群聊评审、多种控制模式 |
| 实时游戏 | AgentScope | MsgHub、分布式支持、结构化输出 |
| 复杂多任务 | CAMEL Workforce | 动态工作者池、智能路由 |

### 核心权衡

1. **涌现式协作 vs 显式控制**：需要根据任务特性选择，复杂流程选择显式控制，探索性任务可选择涌现式
2. **框架灵活性 vs 工程成本**：自定义开发成本高但灵活，框架开发快但有局限
3. **实时性 vs 可审计性**：金融审批等场景优先可审计性，实时交互优先低延迟
