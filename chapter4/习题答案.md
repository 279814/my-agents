# 第四章 智能体经典范式构建 - 习题答案

## 1. 三种经典智能体范式分析

### 1.1 三种范式在"思考"与"行动"的组织方式上的本质区别

**ReAct (Reasoning and Acting)**：
- **核心特点**：思考与行动紧密交织，在循环中交替进行
- **组织方式**：每一步都包含"思考(Thought)-行动(Action)-观察(Observation)"的循环
- **工作流程**：基于上一步的观察结果进行下一步思考，动态调整决策
- **适用场景**：需要根据环境反馈灵活调整、任务路径不明确的动态场景

**Plan-and-Solve**：
- **核心特点**：严格分离思考与行动，先规划后执行
- **组织方式**：规划阶段一次性生成完整的行动计划，执行阶段严格按计划执行
- **工作流程**：静态规划，规划完成后不再修改，按步骤顺序执行
- **适用场景**：有明确步骤、可预测性强、需要全局一致性的任务

**Reflection**：
- **核心特点**：在行动后进行反思和优化，形成迭代改进循环
- **组织方式**："执行-反思-优化"的循环，思考发生在行动之后
- **工作流程**：先生成初步解决方案，然后评估并改进，多次迭代直到满意
- **适用场景**：需要高质量输出、可以迭代优化、对结果有较高要求的任务

### 1.2 智能家居控制助手范式选择

**选择范式**：**Plan-and-Solve** 为主，结合 **Reflection** 机制

**理由**：
1. **任务特性分析**：
   - 智能家居控制涉及多个设备（灯光、空调、窗帘等）
   - 需要根据用户习惯自动调节，有明确的规则和模式
   - 设备间可能存在依赖关系（如先开空调再调温度）

2. **Plan-and-Solve的优势**：
   - 可以一次性生成完整的控制计划，确保设备协同工作
   - 避免ReAct的步进式决策可能导致的设备冲突
   - 适合基于用户习惯的规则化控制

3. **Reflection的补充作用**：
   - 用于优化控制策略，学习用户偏好
   - 通过反思改进控制效果，提升用户体验

### 1.3 范式组合与混合架构设计

**可以组合使用**，不同范式互补优势：

**混合范式架构设计**：

**工作流程**：
1. **任务分类**：根据任务特性选择主导范式
2. **范式协作**：
   - 复杂任务：Plan-and-Solve生成主计划，ReAct处理动态子任务
   - 质量敏感任务：ReAct/Plan-and-Solve生成初稿，Reflection迭代优化
3. **结果整合**：整合各范式输出，确保一致性

**适用场景**：
- **复杂多步骤任务**：Plan-and-Solve规划，ReAct执行
- **高质量输出需求**：ReAct执行，Reflection优化
- **动态环境任务**：ReAct为主，Plan-and-Solve提供高层指导
- **学习优化任务**：执行+Reflection循环，持续改进

## 2. ReAct输出解析机制分析

### 2.1 当前正则表达式解析的潜在脆弱性

**现有解析代码**：
```python
def _parse_output(self, text: str):
    thought_match = re.search(r"Thought:\s*(.*?)(?=\nAction:|$)", text, re.DOTALL)
    action_match = re.search(r"Action:\s*(.*?)$", text, re.DOTALL)
    thought = thought_match.group(1).strip() if thought_match else None
    action = action_match.group(1).strip() if action_match else None
    return thought, action
```

**脆弱性分析**：

1. **格式依赖性强**：
   - 严格依赖"Thought:"和"Action:"关键词
   - 大小写、空格、标点符号差异可能导致匹配失败
   - 如"thought:"、"THOUGHT:"、"Action："可能无法匹配

2. **内容干扰问题**：
   - Thought内容中如果包含"Action:"字符串，会提前截断
   - 如："Thought: 下一步应该Action: 搜索相关信息"会被错误解析

3. **语言和编码问题**：
   - 多语言模型可能使用其他语言关键词
   - 特殊字符、Unicode字符可能导致正则表达式失败

4. **边界情况处理不足**：
   - 没有Action只有Thought的情况处理不完善
   - 多行格式、额外换行符可能影响匹配
   - 模型输出格式偏差（如添加额外说明文字）

5. **错误恢复能力弱**：
   - 匹配失败直接返回None，缺乏降级处理
   - 没有尝试其他解析策略

**失败场景示例**：
- 模型输出："我认为应该先思考。Thought: 我需要搜索\nAction: Search[华为手机]"
- 模型输出："Throught: [拼写错误] 我需要信息\nAct: Search[查询]"
- 模型输出包含Markdown格式："**Thought:** 我的想法\n**Action:** Search[关键词]"

```python
def parse_enhanced(text):
    # 使用更灵活的分隔符匹配
    thought = None
    action = None

    # 尝试多种分隔符格式
    patterns = [
        (r'Thought\s*[:：]\s*(.*?)(?=\s*(?:Action|行动)\s*[:：]|$)',
         r'(?:Action|行动)\s*[:：]\s*(.*?)$'),
        (r'思考\s*[:：]\s*(.*?)(?=\s*(?:行动|Action)\s*[:：]|$)',
         r'(?:行动|Action)\s*[:：]\s*(.*?)$')
    ]

    for thought_pattern, action_pattern in patterns:
        thought_match = re.search(thought_pattern, text, re.DOTALL | re.IGNORECASE)
        action_match = re.search(action_pattern, text, re.DOTALL | re.IGNORECASE)
        if thought_match or action_match:
            thought = thought_match.group(1).strip() if thought_match else None
            action = action_match.group(1).strip() if action_match else None
            break

    return thought, action
```

```python
            thought = root.find('thought').text if root.find('thought') is not None else None
```

|------|------|------|----------|

**推荐策略**：
1. **简单场景**：增强版正则表达式（多模式匹配）
2. **生产环境**：JSON/XML结构化输出
3. **高可靠性要求**：混合解析策略+降级处理
4. **OpenAI生态**：优先使用函数调用

### 3.1 添加"计算器"工具

**工具实现**：
```python
import ast
import math
import operator

class CalculatorTool:
    """安全计算器工具，支持复杂数学表达式"""

    @staticmethod
    def calculate(expression: str) -> str:
        """
        计算数学表达式，支持加减乘除、括号、常用函数
        示例: calculate("(123 + 456) × 789 / 12")
        """
        try:
            # 预处理表达式
            expr = expression.strip()

            # 替换中文符号和常用函数名
            replacements = {
                '×': '*', '÷': '/', '（': '(', '）': ')',
                'π': 'math.pi', 'e': 'math.e',
                'sin': 'math.sin', 'cos': 'math.cos', 'tan': 'math.tan',
                'sqrt': 'math.sqrt', 'log': 'math.log', 'ln': 'math.log',
                '^': '**', '²': '**2', '³': '**3'
            }

            for old, new in replacements.items():
                expr = expr.replace(old, new)

            # 安全评估表达式
            # 限制可用函数和操作符
            allowed_nodes = {
                ast.Expression, ast.BinOp, ast.UnaryOp, ast.Constant,
                ast.Add, ast.Sub, ast.Mult, ast.Div, ast.Pow,
                ast.USub, ast.UAdd  # 一元操作符
            }

            # 解析并验证AST
            tree = ast.parse(expr, mode='eval')

            for node in ast.walk(tree):
                if type(node) not in allowed_nodes:
                    raise ValueError(f"不允许的操作: {type(node).__name__}")

            # 编译并执行
            code = compile(tree, '<string>', 'eval')

            # 限制可用的命名空间
            safe_namespace = {
                '__builtins__': {},
                'math': math,
                'pi': math.pi,
                'e': math.e
            }

            result = eval(code, safe_namespace)

            # 格式化结果
            if isinstance(result, float):
                if result.is_integer():
                    return f"{int(result)}"
                else:
                    # 保留合理精度
                    return f"{result:.6f}".rstrip('0').rstrip('.')
            else:
                return str(result)

        except ZeroDivisionError:
            return "错误: 除以零"
        except ValueError as e:
            return f"错误: {str(e)}"
        except SyntaxError:
            return "错误: 表达式语法错误"
        except Exception as e:
            return f"计算错误: {str(e)}"

# 注册到ToolExecutor
tool_executor = ToolExecutor()

    name="Calculator",
    description="计算数学表达式，支持加减乘除、括号、常用函数如sin/cos/tan/sqrt等",
    func=CalculatorTool.calculate
)
```

### 3.2 工具选择失败处理机制

**机制设计**：
```python
class EnhancedToolExecutor(ToolExecutor):
    """增强的工具执行器，包含错误处理和引导机制"""

    def __init__(self, max_retries=3, error_threshold=2):
        super().__init__()
        self.max_retries = max_retries
        self.error_threshold = error_threshold
        self.error_counter = {}  # 记录每个工具的错误次数
        self.session_errors = []  # 记录本次会话的错误历史

    def execute_with_retry(self, tool_name: str, tool_input: str, agent_context: dict) -> str:
        """
        增强的执行方法，包含重试和错误引导
        """
        # 检查工具是否存在
        if tool_name not in self.tools:
            error_msg = f"工具 '{tool_name}' 不存在。可用工具: {', '.join(self.tools.keys())}"
            self._record_error(tool_name, "tool_not_found")
            return self._generate_guidance(error_msg, agent_context)

        # 检查错误次数是否超过阈值
        if self.error_counter.get(tool_name, 0) >= self.error_threshold:
            return self._suggest_alternative(tool_name, tool_input, agent_context)

        # 执行工具
        for attempt in range(self.max_retries):
            try:
                tool_func = self.tools[tool_name]["func"]
                result = tool_func(tool_input)

                # 检查结果是否有效
                if self._is_valid_result(result):
                    # 重置错误计数器
                    self.error_counter[tool_name] = 0
                    return result
                else:
                    # 结果无效，记录错误
                    self._record_error(tool_name, "invalid_result")

            except Exception as e:
                self._record_error(tool_name, str(e))

            # 如果不是最后一次尝试，提供反馈并重试
            if attempt < self.max_retries - 1:
                feedback = self._generate_feedback(tool_name, attempt + 1)
                # 这里可以将反馈返回给智能体，让智能体调整输入
                return f"第{attempt + 1}次尝试失败: {feedback}. 请调整输入参数后重试。"

        # 所有尝试都失败
        return self._escalate_to_human_or_fallback(tool_name, tool_input, agent_context)

    def _record_error(self, tool_name: str, error_type: str):
        """记录错误"""
        self.error_counter[tool_name] = self.error_counter.get(tool_name, 0) + 1
        self.session_errors.append({
            "tool": tool_name,
            "error": error_type,
            "timestamp": datetime.now().isoformat()
        })

    def _generate_guidance(self, error_msg: str, context: dict) -> str:
        """生成引导信息"""
        guidance_templates = [
            f"操作失败: {error_msg}。请检查工具名称是否正确。",
            f"无法执行: {error_msg}。建议: 1) 确认工具名 2) 检查参数格式 3) 查看可用工具列表",
            f"错误: {error_msg}。当前上下文: {context.get('last_thought', '无')}。是否需要重新思考任务分解？"
        ]
        return random.choice(guidance_templates)

    def _suggest_alternative(self, tool_name: str, input_str: str, context: dict) -> str:
        """建议替代工具"""
        # 基于工具描述进行相似度匹配
        target_description = self.tools[tool_name]["description"]
        alternatives = []

        for name, info in self.tools.items():
            if name != tool_name:
                similarity = self._calculate_similarity(target_description, info["description"])
                if similarity > 0.3:  # 相似度阈值
                    alternatives.append((name, info["description"], similarity))

        if alternatives:
            alternatives.sort(key=lambda x: x[2], reverse=True)
            suggestion = f"工具 '{tool_name}' 多次使用失败。建议尝试: "
            suggestion += ", ".join([f"'{name}' ({desc[:50]}...)" for name, desc, _ in alternatives[:3]])
            return suggestion
        else:
            return f"工具 '{tool_name}' 多次失败，且无类似工具可用。建议重新规划任务。"

    def _calculate_similarity(self, desc1: str, desc2: str) -> float:
        """计算描述文本相似度（简化版）"""
        words1 = set(desc1.lower().split())
        words2 = set(desc2.lower().split())
        if not words1 or not words2:
            return 0.0
        return len(words1 & words2) / len(words1 | words2)

    def _is_valid_result(self, result: str) -> bool:
        """检查结果是否有效"""
        if not result or result.strip() == "":
            return False
        # 检查是否包含错误关键词
        error_indicators = ["错误", "失败", "无效", "无法", "不支持", "error", "fail"]
        for indicator in error_indicators:
            if indicator in result.lower():
                return False
        return True

    def _generate_feedback(self, tool_name: str, attempt: int) -> str:
        """生成具体反馈"""
        tool_desc = self.tools[tool_name]["description"]
        feedback_map = {
            1: f"第一次尝试失败。工具 '{tool_name}' 的功能是: {tool_desc}。请检查参数格式。",
            2: f"第二次尝试失败。请确认输入是否适合工具 '{tool_name}' 的处理范围。",
            3: f"工具 '{tool_name}' 多次失败。建议: 1) 简化输入 2) 分割任务 3) 使用不同工具"
        }
        return feedback_map.get(attempt, "请调整输入后重试。")

    def _escalate_to_human_or_fallback(self, tool_name: str, input_str: str, context: dict) -> str:
        """升级到人工处理或使用备用方案"""
        # 记录需要人工干预
        self.session_errors.append({
            "tool": tool_name,
            "input": input_str,
            "context": context,
            "action": "requires_human_intervention",
            "timestamp": datetime.now().isoformat()
        })

        return f"工具 '{tool_name}' 多次执行失败，已记录并需要人工干预。建议暂停当前任务或尝试其他方法。"
```

### 3.3 工具数量扩展优化方案

**问题分析**：
当工具数量从几个增加到50-100个时，当前简单描述列表方式面临的问题：
1. **描述信息过载**：提示词中包含太多工具描述，超出模型上下文窗口
2. **检索效率低**：模型需要从大量工具中手动选择，容易出错
3. **维护困难**：工具更新、添加、删除需要频繁修改提示词

```python
class VectorizedToolRetriever:
    """基于向量相似度的工具检索"""

    def __init__(self, embedding_model):
        self.embedding_model = embedding_model
        self.tool_embeddings = {}  # 工具名称 -> 向量
        self.tool_descriptions = {}

    def add_tool(self, name: str, description: str):
        """添加工具并生成向量"""
        self.tool_descriptions[name] = description
        self.tool_embeddings[name] = self.embedding_model.encode(description)

    def retrieve_tools(self, query: str, top_k: int = 5) -> list:
        """检索最相关的工具"""
        query_embedding = self.embedding_model.encode(query)

        # 计算相似度
        similarities = []
        for name, tool_embedding in self.tool_embeddings.items():
            similarity = self._cosine_similarity(query_embedding, tool_embedding)
            similarities.append((name, similarity))

        # 排序并返回top-k
        similarities.sort(key=lambda x: x[1], reverse=True)
        return [name for name, _ in similarities[:top_k]]

    def _cosine_similarity(self, vec1, vec2):
        """计算余弦相似度"""
        dot_product = np.dot(vec1, vec2)
        norm1 = np.linalg.norm(vec1)
        norm2 = np.linalg.norm(vec2)
        return dot_product / (norm1 * norm2)
```

```python
class ToolUsageLearner:

            self.success_rates[task_type] = {}

        self.usage_patterns[task_type].append(tool_sequence)

        # 更新成功率
        for tool in tool_sequence:
            if tool not in self.success_rates[task_type]:
                self.success_rates[task_type][tool] = {"success": 0, "total": 0}

            self.success_rates[task_type][tool]["total"] += 1
            if success:
                self.success_rates[task_type][tool]["success"] += 1

    def recommend_tools(self, task_type: str, current_context: str) -> list:
        """推荐工具"""
        if task_type not in self.usage_patterns:
            return []

        # 基于历史模式推荐
        patterns = self.usage_patterns[task_type]

        # 简单实现：返回最常用的工具
        tool_counts = {}
        for pattern in patterns[-10:]:  # 最近10次
            for tool in pattern:
                tool_counts[tool] = tool_counts.get(tool, 0) + 1

        # 按使用频率排序
        sorted_tools = sorted(tool_counts.items(), key=lambda x: x[1], reverse=True)
        return [tool for tool, _ in sorted_tools[:3]]

    def get_tool_success_rate(self, task_type: str, tool_name: str) -> float:
        """获取工具在特定任务上的成功率"""
        if (task_type in self.success_rates and
            tool_name in self.success_rates[task_type]):
            stats = self.success_rates[task_type][tool_name]
            if stats["total"] > 0:
                return stats["success"] / stats["total"]
        return 0.5  # 默认值
```

## 4. Plan-and-Solve范式深入分析

### 4.1 动态重规划机制设计

**问题分析**：
当前Plan-and-Solve生成的计划是静态的，执行过程中遇到问题无法调整。需要设计动态重规划机制。

**动态重规划机制设计**：

```python
class AdaptivePlanner:
    """自适应规划器，支持动态重规划"""

    def __init__(self, llm_client, max_replan_attempts=3):
        self.llm_client = llm_client
        self.max_replan_attempts = max_replan_attempts
        self.replan_count = 0

    def execute_with_replanning(self, question: str, executor: Executor) -> str:
        """带重规划的执行流程"""
        original_plan = self.generate_plan(question)

        for attempt in range(self.max_replan_attempts):
            print(f"\n=== 执行尝试 {attempt + 1} ===")

            try:
                result = self._execute_plan(original_plan, executor, question)

                # 验证结果
                if self._validate_result(result, question):
                    return result

                # 结果验证失败，需要重规划
                print(f"⚠️ 结果验证失败，触发重规划 (尝试 {attempt + 1})")
                original_plan = self._replan(original_plan, result, question, attempt)

            except PlanExecutionError as e:
                # 执行错误，需要重规划
                print(f"⚠️ 执行错误: {e}, 触发重规划")
                original_plan = self._replan_due_to_error(original_plan, str(e), question, attempt)

            self.replan_count += 1

        # 所有重规划尝试都失败
        return self._fallback_solution(question)

    def _execute_plan(self, plan: list, executor: Executor, question: str) -> str:
        """执行计划，监控执行状态"""
        context = {"original_question": question}

        for i, step in enumerate(plan):
            print(f"\n执行步骤 {i+1}/{len(plan)}: {step}")

            try:
                step_result = executor.execute_step(step, context)

                # 检查步骤执行结果
                if self._is_step_failed(step_result):
                    raise PlanExecutionError(f"步骤 {i+1} 执行失败: {step_result}")

                # 更新上下文
                context[f"step_{i+1}_result"] = step_result

                # 检查是否需要提前终止
                if self._should_terminate_early(step_result, context):
                    print("检测到提前终止条件")
                    return self._compile_results(context)

            except Exception as e:
                raise PlanExecutionError(f"步骤 {i+1} 执行异常: {str(e)}")

        # 所有步骤执行完成
        return self._compile_results(context)

    def _replan(self, old_plan: list, failed_result: str, question: str, attempt: int) -> list:
        """基于失败结果进行重规划"""
        replan_prompt = f"""
        原问题: {question}

        原计划: {old_plan}

        执行失败/结果无效的原因: {failed_result}

        这是第{attempt + 1}次重规划尝试。请分析失败原因，并生成一个新的改进计划。

        考虑以下改进策略:
        1. 分解更细的步骤
        2. 调整步骤顺序
        3. 添加验证步骤
        4. 使用不同的方法

        请输出新的计划（Python列表格式）:
        ```
        ["新步骤1", "新步骤2", ...]
        ```
        """

        messages = [{"role": "user", "content": replan_prompt}]
        response = self.llm_client.think(messages)

        # 解析新计划
        new_plan = self._parse_plan_from_response(response)

        print(f"🔄 生成新计划: {new_plan}")
        return new_plan

    def _replan_due_to_error(self, old_plan: list, error: str, question: str, attempt: int) -> list:
        """因执行错误进行重规划"""
        replan_prompt = f"""
        原问题: {question}

        原计划: {old_plan}

        执行错误: {error}

        错误分析: 请分析错误原因，并生成一个避免此错误的新计划。

        改进建议:
        1. 避免导致错误的操作
        2. 添加错误处理步骤
        3. 使用更安全的方法
        4. 分解复杂操作为简单步骤

        请输出新的计划:
        ```
        ["新步骤1", "新步骤2", ...]
        ```
        """

        messages = [{"role": "user", "content": replan_prompt}]
        response = self.llm_client.think(messages)

        return self._parse_plan_from_response(response)

    def _validate_result(self, result: str, question: str) -> bool:
        """验证结果是否合理"""
        validation_prompt = f"""
        问题: {question}

        得到的答案: {result}

        请判断这个答案是否合理、完整、正确。
        输出格式: VALID 或 INVALID

        如果VALID，请说明理由。
        如果INVALID，请指出问题。
        """

        messages = [{"role": "user", "content": validation_prompt}]
        response = self.llm_client.think(messages)

        return "VALID" in response.upper()

    def _is_step_failed(self, step_result: str) -> bool:
        """检查步骤是否失败"""
        failure_indicators = ["错误", "失败", "无效", "无法", "不支持", "exception", "error", "fail"]
        return any(indicator in step_result.lower() for indicator in failure_indicators)

    def _should_terminate_early(self, step_result: str, context: dict) -> bool:
        """检查是否需要提前终止"""
        # 示例：如果已经得到最终答案，可以提前终止
        early_answer_indicators = ["答案是", "结果为", "最终答案", "结论是"]
        return any(indicator in step_result for indicator in early_answer_indicators)

    def _compile_results(self, context: dict) -> str:
        """编译最终结果"""
        # 从上下文中提取所有步骤结果
        results = []
        for key in sorted(context.keys()):
            if key.startswith("step_") and key.endswith("_result"):
                results.append(context[key])

        # 简单合并，实际可能需要更复杂的逻辑
        return "\n".join(results)

    def _fallback_solution(self, question: str) -> str:
        """降级解决方案"""
        return f"经过{self.max_replan_attempts}次重规划尝试后仍无法解决问题: {question}。建议: 1) 简化问题 2) 提供更多上下文 3) 人工干预。"
```

**重规划触发条件**：
1. **步骤执行失败**：工具返回错误，异常抛出
2. **结果验证失败**：结果不符合预期或无效
3. **上下文不一致**：步骤结果与后续步骤要求冲突
4. **性能超时**：步骤执行时间过长
5. **资源限制**：内存、网络等资源不足

**重规划策略**：
1. **局部调整**：只修改失败步骤及其相关步骤
2. **全局重规划**：完全重新生成计划
3. **步骤细化**：将复杂步骤分解为更简单的子步骤
4. **替代方法**：使用不同的方法实现相同目标
5. **添加验证**：在关键步骤后添加验证步骤

### 4.2 旅行预订任务范式对比

```python
class MultiModelReflectionAgent:
    """多模型反思智能体"""

    def __init__(self, execution_model, reflection_model):
        """
        execution_model: 执行模型（快速、经济）
        reflection_model: 反思模型（强大、准确）
        """
        self.execution_model = execution_model
        self.reflection_model = reflection_model
        self.memory = Memory()

    def execute_with_reflection(self, task: str, max_iterations: int = 3) -> str:
        """使用不同模型进行执行和反思"""

        for iteration in range(max_iterations):
            print(f"\n=== 迭代 {iteration + 1} ===")

            # 1. 执行阶段（使用执行模型）
            print("执行阶段...")
            execution_result = self._execute(task, self.execution_model)
            self.memory.add("execution", execution_result)

            # 2. 反思阶段（使用反思模型）
            print("反思阶段...")
            reflection_feedback = self._reflect(
                execution_result,
                task,
                self.reflection_model
            )
            self.memory.add("reflection", reflection_feedback)

            # 3. 检查终止条件
            if self._should_terminate(reflection_feedback, iteration):
                print("达到终止条件")
                return self._compile_final_result()

            # 4. 优化阶段（使用执行模型，根据反思反馈优化）
            print("优化阶段...")
            task = self._generate_optimized_task(task, reflection_feedback)

        return self._compile_final_result()

    def _execute(self, task: str, model) -> str:
        """使用指定模型执行"""
        # 根据不同模型特性调整提示词
        if model == "fast_model":
            prompt = self._create_fast_execution_prompt(task)
        else:
            prompt = self._create_standard_execution_prompt(task)

        return model.think([{"role": "user", "content": prompt}])

    def _reflect(self, execution_result: str, original_task: str, model) -> str:
        """使用指定模型反思"""
        # 反思模型可以使用更复杂的提示词
        reflection_prompt = f"""
        你是一位严格的评审专家。请评估以下执行结果:

        原始任务: {original_task}

        执行结果: {execution_result}

        请从以下维度进行评估:
        1. 正确性: 结果是否准确解决了任务？
        2. 完整性: 是否遗漏了重要方面？
        3. 质量: 解决方案的质量如何？
        4. 效率: 方法是否高效？
        5. 改进建议: 具体如何改进？

        如果无需改进，请明确说明"无需改进"。
        """

        return model.think([{"role": "user", "content": reflection_prompt}])
```

**不同组合的影响**：

| 组合方案 | 优势 | 劣势 | 适用场景 |
|----------|------|------|----------|
| **同模型** | 一致性高，实现简单 | 无法发挥不同模型优势 | 通用场景，资源有限 |
| **强反思+快执行** | 反思质量高，执行成本低 | 反思成本高，可能过度优化 | 质量敏感型任务 |
| **快反思+强执行** | 快速迭代，执行质量高 | 反思可能不够深入 | 效率优先任务 |
| **专业模型组合** | 针对任务优化 | 系统复杂，需要调优 | 专业领域任务 |

**模型选择策略**：

1. **基于任务复杂度**：
   - 简单任务：同模型或快反思+快执行
   - 复杂任务：强反思+强执行或强反思+快执行

2. **基于质量要求**：
   - 高精度要求：强反思模型
   - 一般要求：平衡组合

3. **基于成本约束**：
   - 低成本：快执行+快反思
   - 中等成本：强反思+快执行
   - 高成本：强反思+强执行

4. **基于领域特性**：
   - 创意任务：强执行（生成）+强反思（评估）
   - 分析任务：强反思（分析）+快执行（整理）
   - 编程任务：强执行（编码）+强反思（代码审查）

```python
    """智能终止条件判断"""

    def __init__(self):
        self.improvement_history = []
        self.convergence_threshold = 0.05  # 改进度小于5%认为收敛
        self.max_no_improvement_iterations = 2

    def should_terminate(self,
                        current_result: str,
                        previous_result: str,
                        reflection_feedback: str,
                        iteration: int,
                        max_iterations: int) -> bool:
        """判断是否应该终止"""

        termination_conditions = [
            self._no_improvement_needed(reflection_feedback),
            self._reached_max_iterations(iteration, max_iterations),
            self._quality_converged(current_result, previous_result),
            self._detected_oscillation(current_result),
            self._satisfied_quality_threshold(current_result),
            self._time_or_resource_constraint()
        ]

        return any(termination_conditions)

    def _no_improvement_needed(self, feedback: str) -> bool:
        """反馈明确表示无需改进"""
        no_improvement_phrases = [
            "无需改进", "无需修改", "已经很好了", "足够优秀",
            "no improvement needed", "good enough", "satisfactory"
        ]

        feedback_lower = feedback.lower()
        return any(phrase in feedback_lower for phrase in no_improvement_phrases)

    def _reached_max_iterations(self, iteration: int, max_iterations: int) -> bool:
        """达到最大迭代次数"""
        return iteration >= max_iterations

    def _quality_converged(self, current: str, previous: str) -> bool:
        """质量收敛判断"""
        if not previous:
            return False

        # 计算改进度
        improvement = self._calculate_improvement(current, previous)
        self.improvement_history.append(improvement)

        # 检查最近几次改进是否都很小
        if len(self.improvement_history) >= self.max_no_improvement_iterations:
            recent_improvements = self.improvement_history[-self.max_no_improvement_iterations:]
            avg_improvement = sum(recent_improvements) / len(recent_improvements)

                print(f"质量收敛，平均改进度: {avg_improvement:.2%}")
```

```python
        """确定改进优先级"""
        priorities = []

        for dimension, evaluation in evaluations.items():
            score = evaluation.get("score", 0)
            if score < 0.6:  # 低分维度优先
                priorities.append({
                    "dimension": dimension,
                    "priority": "high",
                    "reason": evaluation.get("summary", "低分")
                })
            elif score < 0.8:
                priorities.append({
                    "dimension": dimension,
                    "priority": "medium",
                    "reason": "有改进空间"
                })

        # 按优先级排序
        priority_order = {"high": 0, "medium": 1, "low": 2}
        priorities.sort(key=lambda x: priority_order[x["priority"]])

        return priorities
        ```

**服务于各自范式的设计逻辑**：

1. **ReAct设计逻辑**：
   - **交互性**：提示词设计为多轮对话格式，支持"思考-行动-观察"循环
   - **工具集成**：明确工具调用语法，使模型学会在适当时候调用工具
   - **错误恢复**：通过历史记录让模型从错误中学习调整
   - **格式强制**：严格的话述格式确保代码可解析执行

2. **Plan-and-Solve设计逻辑**：
   - **一次性规划**：提示词鼓励一次性生成完整计划，避免中途修改
   - **结构清晰**：Python列表格式确保计划易于解析和执行
   - **抽象层次**：强调步骤分解而非具体实现，保持规划的高层次性
   - **确定性**：减少模糊性，确保生成的计划明确可执行

## 7. 电商客服智能体系统设计

### 7.1 系统架构选择

**需求分析**：
- 多步骤任务：理解申请→查询信息→判断决策→生成回复→发送邮件
- 决策逻辑：需要基于公司政策和用户信息进行判断
- 质量要求：决策需合理，回复需得体
- 安全要求：需要自我反思和置信度评估
- 扩展性：未来可能增加更多功能

**架构选择**：**混合范式架构**

```python
class CustomerServiceAgent:
    """电商客服智能体 - 混合范式架构"""

    def __init__(self):
        # 核心组件
        self.planner = Planner()          # Plan-and-Solve: 任务分解
        self.executor = ReactExecutor()   # ReAct: 具体执行
        self.reflector = Reflector()      # Reflection: 自我评估
        self.tool_manager = ToolManager() # 工具管理

        # 配置
        self.confidence_threshold = 0.7   # 置信度阈值

    def handle_refund_request(self, user_request: str, user_id: str) -> dict:
        """处理退款请求 - 主流程"""

        # 阶段1: 规划分解 (Plan-and-Solve)
        plan = self.planner.create_plan(user_request)
        # ["理解用户意图", "查询订单信息", "检查物流状态",
        #  "评估退款资格", "生成回复草稿", "自我评估决策"]

        results = {}
        decision_confidence = 1.0

        # 阶段2: 逐步执行 (ReAct)
        for step in plan:
            if step == "理解用户意图":
                results["intent"] = self.executor.understand_intent(user_request)

            elif step == "查询订单信息":
                results["order_info"] = self.tool_manager.query_order(user_id)

            elif step == "检查物流状态":
                results["logistics"] = self.tool_manager.check_logistics(
                    results["order_info"]["order_id"]
                )

            elif step == "评估退款资格":
                decision_result = self.executor.evaluate_refund(
                    results["intent"],
                    results["order_info"],
                    results["logistics"]
                )
                results["decision"] = decision_result["decision"]
                decision_confidence = decision_result["confidence"]

                # 置信度检查
                if decision_confidence < self.confidence_threshold:
                    # 触发反思
                    reflection = self.reflector.reflect_decision(
                        results, decision_confidence
                    )
                    results["reflection"] = reflection

                    # 基于反思重新评估
                    revised_decision = self.executor.revise_decision(
                        results, reflection
                    )
                    results["revised_decision"] = revised_decision
                    decision_confidence = revised_decision["confidence"]

            elif step == "生成回复草稿":
                results["reply_draft"] = self.executor.generate_reply(
                    results.get("revised_decision", results["decision"]),
                    user_request
                )

            elif step == "自我评估决策":
                final_evaluation = self.reflector.final_check(
                    results, decision_confidence
                )
                results["final_evaluation"] = final_evaluation

        # 阶段3: 执行最终动作
        if results.get("final_evaluation", {}).get("proceed", True):
            # 发送邮件
            email_result = self.tool_manager.send_email(
                user_id,
                results["reply_draft"]
            )
            results["email_sent"] = email_result

        return results
```

**架构优势**：
1. **Plan-and-Solve**：确保处理流程完整、有序
2. **ReAct**：灵活处理各种查询和判断
3. **Reflection**：保证决策质量，降低风险
4. **模块化设计**：各组件职责清晰，易于维护扩展

### 7.4 风险分析与技术缓解

**主要风险**：

1. **决策错误风险**：
   - **表现**：错误批准或拒绝退款，导致财务损失或客户流失
   - **缓解**：
     - 多级置信度检查，低置信度转人工
     - 决策日志和审计追踪
     - 定期模型评估和更新
     - A/B测试验证决策效果

2. **安全与合规风险**：
   - **表现**：数据泄露、未授权访问、违反隐私法规
   - **缓解**：
     - 严格的数据访问控制
     - 敏感信息脱敏处理
     - 合规性检查集成
     - 安全审计和监控

3. **模型幻觉风险**：
   - **表现**：生成虚假信息或错误引用政策
   - **缓解**：
     - 事实核查机制
     - 政策数据库准确查询
     - 输出验证和限制
     - 人工审核关键决策

4. **用户体验风险**：
   - **表现**：回复生硬、不理解用户情绪、沟通不畅
   - **缓解**：
     - 情感分析集成
     - 多轮对话优化
     - 用户反馈收集
     - 持续对话质量监控

5. **系统可扩展性风险**：
   - **表现**：流量高峰性能下降、新功能集成困难
   - **缓解**：
     - 微服务架构
     - 弹性伸缩设计
     - 清晰的API接口
     - 模块化工具系统

6. **伦理与公平性风险**：
   - **表现**：决策偏见、不公平对待用户
   - **缓解**：
     - 公平性检测算法
     - 多样化训练数据
     - 伦理审查流程
     - 透明度报告

**具体实施措施**：

1. **多层审核机制**：
   ```python
   class MultiLevelReview:
       def review_decision(self, decision: dict, context: dict) -> dict:
           # 第一层: 自动规则检查
           if not self._rule_check(decision, context):
               return {"action": "reject", "reason": "规则检查失败"}

           # 第二层: 置信度检查
           if decision["confidence"] < self.low_confidence_threshold:
               return {"action": "human_review", "priority": "high"}

           # 第三层: 风险模式匹配
           risk_score = self._risk_assessment(decision, context)
           if risk_score > self.risk_threshold:
               return {"action": "human_review", "priority": "medium"}

           # 第四层: 随机抽样审计
           if random.random() < self.audit_sample_rate:
               return {"action": "audit", "add_to_audit_queue": True}

           return {"action": "approve", "auto_approved": True}
   ```

2. **持续优化机制**：
   - **反馈学习循环**：收集用户反馈改进模型
   - **A/B测试框架**：测试不同决策策略的效果
   - **性能监控**：实时监控系统各项指标
   - **定期评估**：定期评估系统效果和风险

3. **透明度与可解释性**：
   - 记录完整的决策链条
   - 提供决策依据的可视化解释
   - 允许用户查询决策理由
   - 定期发布系统性能报告

**上线建议**：
1. **分阶段上线**：先小范围测试，逐步扩大
2. **影子模式运行**：对比智能体与人工决策
3. **逐步放权**：从辅助角色逐步过渡到主要处理
4. **持续监控**：建立完善的监控和告警系统
5. **应急计划**：准备手动接管和回滚方案

通过上述设计，电商客服智能体能够在保证决策质量和用户体验的同时，有效控制各种风险，实现降本增效的目标。
